## 1) Target architecture (for “highly editable”)

Use 5 separable components:

1. **Config surface**  
   Add MXFP8 switches in `Qwen3Config` (enable/disable, block size, format variant, accum dtype, target modules).

2. **Quantization primitives (pure PyTorch reference)**  
   Independent functions/classes for:
   - `quantize_mxfp8(x, scale_meta, ...)`
   - `dequantize_mxfp8(q, scale_meta, ...)`
   - optional blockwise scale policy

3. **Quantized module wrappers**  
   Replace `nn.Linear` in attention/MLP with `MXFP8Linear` wrappers that are easy to inspect and modify.

4. **Runtime control context (future-proof for MSB→LSB)**  
   A mutable context object passed via `**kwargs` through model/layer/attention:
   - stores intermediate states
   - exposes callbacks/hooks
   - can influence later row/column/layer compute

5. **Model patcher / injection utility**  
   One function to transform a `Qwen3Model` instance in-place (or at init) by replacing selected modules with MXFP8 wrappers.

This separation keeps research logic editable without breaking core model logic.
